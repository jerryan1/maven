3.6 pom.xml详解
version 定义规则
主版本号：表示了项目的重大架构变更  
次版本号：表示较大范围的功能添加和变化 
增量版本号：一般表示重大Bug修复  
snapshot 快照
alpha 内测
beta 公测
release 稳定版本
GA 最稳定版本
Final:正式版 
Pro(professional):专业版 
Plus:加强版 
Retail:零售版 
DEMO：演示版
Build:内部标号 
Delux:豪华版  (deluxe: 豪华的，华丽的)
Corporation或Enterprise企业版 
M1 M2 M3  M是milestone的简写 里程碑的意思
RC版本  RC：(Release Candidate)  RC版本，几乎就不会加入新的功能了，而主要着重于除错
SR:修正版 
Trial:试用版
Plus:加强版 
Shareware共享版
Pro(professional):专业版 
Full:完全版 
  
<properties></properties>统一设置版本号，${取值}
<dependencyManagement></dependencyManagement>
dependencyManagement用于定义共同的依赖jar包，用于其他项目继承时使用的是同一版本jar包
